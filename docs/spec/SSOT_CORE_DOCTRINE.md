## 1. コア・ファンタジー（不変条件 / Invariants）

### I-1: 世界は「監視」ではなく「手に余っている」

- 企業（組織）は万能ではない。管理は"統制"ではなく **弛緩した運用（loose governance）**。
- 通信・ログ・デバッグ痕跡は、プレイヤーを縛る鎖ではなく **帰還可能性の手触り（breadcrumb of returnability）** として機能する。

### I-2: プレイヤーの一次快楽は「世界が返事をする」こと

- 本作の快楽は「調べると説明が出る」ではなく、**作用 → 反作用 → 収束の手触り（closed-loop interaction）**。
- したがってチュートリアルは **説明** ではなく **現象学（phenomenology）** で行う。

### I-3: 会社の不条理は"暴力的支配"ではなく"制度の惰性"

- 上司・制度は苛烈な監視者というより、KPI 化した言葉だけが先に増殖している状態。
- 不条理は「罰」ではなく **「規格のズレ」「運用の残骸」** として滲む。

### I-4: 反ストリーマー要素は「直球の憎悪」ではなく"外部性の設計問題"として扱う

- ストリーマーへの憎悪は、主旋律にするとプレイヤーの同調圧を上げる。したがって **本編の基調は中立**。
- ただし"ポートフォリオとしての挑発章"は保持する：**本編と切り離せるモジュール（Chapter / Modifier）** として実装可能にしておく。

## 2. オンボーディング設計（可変ダイヤル / Dials）

ここは「秒数の断定」ではなく **予算（budget）** として持つ。

### D-1: Time-to-Agency（操作可能になるまで）

- **目標レンジ**: 60 -- 120 秒
- 60 秒を切ると"狼狽死"が増え、120 秒を超えると"儀式疲れ"が増える（※どちらも許容、ただし意図を明示）。
- OP ムービーは「最初に強制」ではなく、**起動直後の分岐（即開始 / 即続行 / 後追い視聴）** で吸収する。

### D-2: First Reciprocity（最初の"世界の返事"）

- **目標レンジ**: 操作開始から 30 -- 60 秒以内に、世界が反射的に返事をする。
- 具体の形は固定しない：
  - 例）投げたものが弾かれる / 別物に変換される / 軌道が歪む / 音として返ってくる 等
- 重要なのは「返ってきたものを掴める」ではなく、**返報（feedback）を"回収可能な資源"に変える感覚**。

### D-3: Readability（読み物の共存）

- 読み物は "止まって読む" だけにしない。
  - **移動中に拾える短文（microtext）** と
  - **安全域で腰を据えて読む長文（macrotext）** を二層化する。
- UI は "増やす/減らす" ではなく、**読ませる時だけ視界を整える（視覚ノイズを一時的に落とす）** 方向で。

## 3. レベル設計の指針（不変条件 + 実装への落とし）

### I-5: 遠景は「登りたい山」、近景は「触りたい違和感」

- **遠方目標（distal lure）** は山のように、移動意志を発火させる輪郭を持つ。
- **近景目標（proximal cue）** は浮遊する輝石のように、「ああ、あれね」と理解できるだけの文法を持つ。
- つまり、プレイヤーの注意を"箱に吸わせる"のではなく、**注意の配分を設計する**。

### I-6: インタラクトは"箱を置けば調べる"という希望的観測から降りる

- 「調べさせる」は弱い。代わりに
  - **作用させたくなる配置（affordance）** と
  - **作用した結果の差分（state change）** を設計する。
- "調べる"は補助線に留め、主線は **作用して変える**。

## 4. 広告（Ad）システムの扱い（未決定の正しい保存）

広告は、ここで決め打ちしない。代わりに **二態（modes）** として保存する。

### Mode A: 実広告（収益化）

- F2P 前提、または有料＋広告の異様さが主題になる場合のみ採用。

### Mode B: 架空広告（記号論的 Ad）

- "アド＝アドバンテージ"として、プレイヤーの失敗に追従して世界に増殖する「制度的ノイズ」として扱う。

> **Invariant**: 広告は UI でなく **世界の現象として見える**。収益の有無だけが可変。

## 5. ドキュメント規約（硬直化を防ぐための SSOT）

### R-1: 断定の形式を分離する

下位文書は、記述を必ず以下に分ける。

| ラベル | 意味 | 例 |
|--------|------|-----|
| **Invariant** | 破ってはいけない | 「物理ベースで制御する」 |
| **Dial** | 調整幅（予算） | 「FOV 70--80°」 |
| **Instance** | 現時点の仮の具現 | 「FOV = 75° (Phase 2A)」 |
| **Test** | 採否の判定基準 | 「壁抜け 0 件 / 30 秒プレイ」 |

### R-2: 禁止するのは「比喩」ではなく「比喩のまま放置」

- 「○○のような」は禁止しない。
- 禁止するのは **「○○のような」で終わって、測定可能な差分に落ちない** こと。

## 6. "挑発章（ポートフォリオ）"の格納方法

- 反ストリーマー / 社内制度の悪意は **本編から切れるようにする**。
- 実装上は「Chapter Modifier」または「Result Screen Skin」として導入し、本編の倫理基調を破壊しない。

| 文書 | 旧ステータス | 新ステータス | 備考 |

| SKILL_IDEAS.md | アイデア集 | **アイデア・プール**（変更なし） | — |

### GDD1.0 の主な降格理由

1. **"言い回し禁止"が硬直化を誘発** — 比喩や包括表現を封じ、本 SSOT §R-2 と衝突する。
2. **オンボーディングの断定が強すぎる** — 「5 分以内にフルサイクル」「テキスト説明一切なし」等が設計自由度を奪う。本 SSOT §D-1/D-3 の budget 方式へ移行。
3. **ストリーマーを"拡散装置"として前提化** — 挑発章のモジュール化（§6）と逆向き。
4. **上司ポップアップが"監視/干渉"に寄りすぎ** — 本 SSOT §I-1/I-3 の「弛緩した運用」「制度の惰性」温度感と矛盾。
5. **バイオーム 3 種"厳格"限定** — 遠景誘惑や近景文法の多様性を早期に閉じるリスク。Dial に置くべき。

### PLAYER_CONTROL_SYSTEM の注意点

- 具体的な入力割当やモード定義が密で優秀な一方、SSOT 扱いすると「ボタン配置は 1 つ」系の硬直に繋がりやすい。
- **Instance（現段階の標準案）** としてラベルを付け直す。

## 8. 用語集（ミニ辞典）

| 用語 | 定義 |
|------|------|
| **SSOT** (Single Source of Truth) | 仕様の最上位参照点。下位文書の矛盾を裁く"法源" |
| **Invariant / Dial / Instance / Test** | 硬直化を避けるための記述分離。断定を"型"で管理する |
| **Diegesis（ダイエジェシス）** | 物語世界内に属する表現。UI や広告を世界の現象として扱う時に重要 |
| **Affordance（アフォーダンス）** | 見ただけで"したくなる"行為可能性。箱を置くより強い誘導 |
| **Phenomenology（現象学）** | 説明ではなく体験の差分で理解させる設計態度 |
| **Closed-loop interaction** | 作用 → 反作用 → 回収/転用で"世界の法則"を身体化させる |
| **Distal lure / Proximal cue** | 遠景の移動目標と近景の触発トリガーの二層誘引 |
