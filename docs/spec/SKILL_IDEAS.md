## 1. スキルシステム概要

### コンセプト: 「物理法則の改変」

プレイヤーは猫＋ドローンの組み合わせにより、通常のキャラクターでは不可能な物理法則の「ハック」を獲得していく。スキルツリーはスフィア盤（FF10 のような球状グリッド）を想定し、複数の経路から異なるビルドを構築可能にする。

### スフィア盤の軸（仮）

```
           [重力制御]
              │
              │
[速度制御] ──●── [空中制御]
              │
              │
           [時間制御]
```

| 軸 | テーマ | 例 |
|----|--------|-----|
| **速度制御** | 地上移動の強化 | ダッシュ強化、加速度アップ、急制動 |
| **重力制御** | 落下・ジャンプの改変 | Fast Fall 強化、多段ジャンプ、滞空 |
| **空中制御** | 空中での自由度 | エアダッシュ、空中旋回、グライド |
| **時間制御** | スローモーション系 | バレットタイム延長、時間停止（短時間） |

### 開放リソース（仮）

| リソース名 | 入手方法 | 備考 |
|-----------|---------|------|
| **Glitch Core** | AI Prop を Normalized にすることで獲得 | メインのスキルポイント |
| **Rare Fragment** | 特定条件の達成（隠しエリア、タイムアタック等） | レアスキル用 |

## 3. スキルツリー構成案（スフィア盤レイアウト）

```
                        [T-003 Time Stop]
                              │
                       [T-001 Extended Slow]
                              │
                       [T-002 Quick Reflex]
                              │
    [S-005 Drift Brake]──[S-001 Dash Boost]──[G-001 Heavy Drop]
           │                   │                     │
    [S-006 Momentum]     ●  START  ●           [G-002 Impact Wave]
           │                   │                     │
    [S-002 Dash Turn]──[A-003 Air Steer]───[G-003 Feather Fall]
           │                   │                     │
    [S-003 Double Dash]  [A-001 Air Dash]      [G-004 Double Jump]
           │                   │                     │
    [S-004 Rush Hour]    [A-002 Glide]         [G-005 Wall Jump]
                               │                     │
                         [A-004 Hover]         [G-006 Gravity Flip]
                               │
                         [A-005 Grapple Launch]
```

> **注意**: これは概念的なレイアウトです。実際のスフィア盤は2D/3Dのノードグラフとして実装し、各ノード間の接続はデータドリブン（ScriptableObject）で定義します。

## 4. アイデア追加テンプレート

新しいスキルアイデアを追加する際は、以下のフォーマットを使用してください。

```
| # | スキル名 | 効果 | ステータス | メモ |
|---|---------|------|-----------|------|
| X-NNN | **名前** | 効果の説明 | `[検討中]` | 備考・着想元・バランス懸念など |
```

**ステータスの定義**:
- `[検討中]` — アイデア段階。実装未定。
- `[採用]` — 実装が決定。PLAYER_CONTROL_SYSTEM.md に転記済み。
- `[保留]` — 良いアイデアだが、現フェーズでは見送り。理由をメモに記載。
- `[却下]` — 不採用。理由をメモに記載。将来の参考のため削除しない。
