## 1. エグゼクティブ・サマリー

### 1.1. ワンライン・ピッチ

> **「予算不足のAIディストピアで、猫の社畜がバグだらけの世界を物理で殴って掃除する」**

### 1.2. コアテーマ

3D生成AIが構築した不気味で広大な仮想空間。ブラック企業の極端なコスト削減により、世界の大半が安価で不安定なAI生成アセットで覆われている。プレイヤーは現場のエリートデバッガー（と思い込んでいる猫）となり、壊れた世界を「物理的に」修正しながら、人間製アイテムという「人間性の欠片」を回収する。

### 1.3. 3つのコア体験

| 柱 | 体験 | 要約 |
|----|------|------|
| **Fix the Glitch** | 整地・掃除の快感 | 不気味にうごめくAI世界を暴力とギミックで「定常化」する |
| **逆チューリングテスト** | 鑑定パズルの知的快感 | 完璧すぎるAI生成物の中から、歪な人間製アイテムを見つけ出す |
| **労働の不条理** | 不便益のユーモア | 効率のために車を空へ射出する狂った社内ルールに従うストレスを笑いに転換 |

### 1.4. 禁止事項（Anti-Patterns）

- **NG**: 「○○のような」という曖昧な類似定義。本書の用語と仕様で語ること。
- **NG**: プレイヤーへの過度なナラティブの押し付け。ストーリーは環境と独り言で語る。
- **NG**: ゲージ管理によるサン値/正気度システム。死は物理的かつ即物的に描く。
- **NG**: テキスト説明によるチュートリアル。ルールは「現象」で教える（§6.1参照）。

## 2. ゲームプレイ・メカニクス

### 2.1. コアループ

```
┌─────────────────────────────────────────────────┐
│  探索 (Scout)                                    │
│  ├─ デバッグ視界でエリアをスキャン               │
│  ├─ AI Prop / Human Prop を識別                  │
│  └─ ルート・危険度を判断                         │
│                    ↓                              │
│  制圧 (Siege)                                    │
│  ├─ 要塞のシールド装置を「缶蹴り」で弾き飛ばす  │
│  ├─ エリアを定常化（正常化）                     │
│  └─ AI Prop → 静止オブジェクト化                 │
│                    ↓                              │
│  収集 (Loot)                                     │
│  ├─ Human Prop / コンポーネントを回収            │
│  ├─ 逆チューリングテスト: 本物を見極める         │
│  └─ インベントリ・テトリスで荷物整理             │
│                    ↓                              │
│  帰還・納品 (Submit)                             │
│  ├─ 物理バグ利用のファストトラベルで帰還         │
│  ├─ 納品ゲート(Upload Port)にノルマを投入        │
│  └─ 業務日報形式のリザルト → 報酬精算            │
│                    ↓                              │
│  拡張 (Expand)                                   │
│  ├─ コンポーネントでビルド構築・装備強化         │
│  ├─ 新エリアへのアクセス解放                     │
│  └─ 次のミッションへ ─────────→ [探索] へ戻る    │
└─────────────────────────────────────────────────┘
```

### 2.2. 探索とデバッグ（スキャン）

#### デバッグ視界（Debug View）
- トグルで起動。視界がワイヤーフレーム描画に切り替わる。
- オブジェクトの「当たり判定」「AI識別ID」「コンポーネントスロット数」が可視化される。
- Human Prop は暖色のハイライト、AI Prop は寒色のハイライトで表示。

#### ドローンビーム（攻撃/インタラクション）
- プレイヤーキャラクター（猫）はアイテムに直接触れない。
- 随伴ドローンがトラクタービームで保持・投擲・攻撃を行う。
- AI Prop にビームで Human Prop を投げつける、または直接ビームを照射して定常化する。
- Human Prop を武器として投擲可能だが、耐久（価値）が減少する。

> **設計意図**: ハンドIK実装の全廃によるコスト削減。かつ「猫が現場監督としてドローンに作業させるサボり」の演出。

### 2.3. 拠点攻略（Siege）

#### シールド装置と缶蹴りメカニクス
- AI Propが群れで守る「要塞」の中心にシールド発生装置がある。
- シールド装置は**破壊不能**。「缶蹴り」の要領で物理的に弾き飛ばす必要がある。
- 弾き飛ばすと一定範囲のAI Propが一斉に定常化（静止オブジェクト化）する。
- 定常化したAI Propは足場・壁・遮蔽物として利用可能になる。

#### AI Propの敵対行動パターン

| 性格タグ | 行動 | 対処法 |
|----------|------|--------|
| 排他的（Exclusive） | 突き飛ばし・落下誘発 | タイミングを読んで回避、または重量物で抑え込む |
| 粘着（Sticky） | 拘束・移動妨害 | ビームで引き剥がす、爆発的物理で吹き飛ばす |
| 擬態（Mimic） | 安全なオブジェクトに偽装 | デバッグ視界で識別、先制攻撃 |
| 集団（Swarm） | 群れで包囲 | 範囲攻撃、シールド装置への直行ルート確保 |

### 2.4. アセットの属性体系

| 種類 | 外見（休眠時） | 外見（敵対/活性時） | 物理挙動 | プレイヤーへの影響 |
|------|---------------|-------------------|----------|------------------|
| **AI Prop** | 不気味に完璧。テクスチャが綺麗すぎ、ポリゴンが均一すぎる。「作り物」の違和感。 | グリッチ崩壊。頂点揺らぎ、色収差、融合、うねり。 | 不安定。接近で攻撃/逃走/擬態。 | 攻撃対象。定常化後は静止オブジェクト（足場/壁）になる。 |
| **Human Prop** | 手作り感。わずかに歪、テクスチャに筆跡、ポリゴンに個性。温かみがある。 | （常に安定） | 安定。物理法則に従う。壊れない。コンポーネントスロットが多い。 | 収集・活用対象。投擲可能だが耐久（価値）が減る。ギミック起動キーにもなる。 |
| **コンポーネント** | 小型の機能パーツ | — | 安定 | アイテムに装着する強化パーツ（重力軽減、発光、スキャン拡張等） |

> **設計意図（矛盾解消）**: AI Propに「二態」を持たせることで、「高精細だがゴミ」（休眠時の不気味な完璧さ）と「グリッチで歪む」（敵対時の視覚フィードバック）を両立させた。Human Propは「不格好だが宝」を採用し、逆チューリングテスト（AI完璧品の中から人間の手作り品を見抜く）の成立を担保する。

### 2.5. インベントリシステム（Inventory Tetris）

#### 基本構造
- **グリッドベースの疑似物理**を採用。見た目上はアイテムがうごめくが、内部データはグリッド上のセル管理。
- バッグのグリッドサイズはアップグレード可能。

#### アイテム挙動

| アイテム種 | インベントリ内挙動 |
|-----------|------------------|
| Human Prop | 固定ブロック。配置後は動かない。安定。 |
| AI Prop（未浄化） | セルを奪い合う。一定間隔で隣接セルへ移動しようとする。Human Propを押し出そうとする。 |
| コンポーネント | 1セル占有。安定。アイテムへの装着時はスロットUIへ移動。 |

#### パズル要素
- AI Propの妨害をかわしながら、限られたスペースにHuman Propを確保する戦略的パズル。
- AI Propを浄化（定常化）するとインベントリ内でも安定するが、「素材」としての価値は下がる。

> **技術的根拠**: 3D物理をコンテナ内で運用するとトンネリング（壁抜け）が頻発する。グリッド制により堅牢性を確保しつつ、視覚的な演出で物理感を維持する。

### 2.6. 成長システム（Progression）

#### コンポーネント・ビルド
- 回収したHuman Propには空きスロット（ソケット）がある。
- 要塞攻略で得たコンポーネントを装着し、自分だけの「最強のガジェット」を構築する。

| コンポーネント例 | 効果 |
|----------------|------|
| 重力キャンセラー | 装着アイテムの重量を無視。巨大Propも軽々投擲可能 |
| スキャン・アンプ | デバッグ視界の範囲拡大。遠距離からHuman Propを識別 |
| AI妨害シールド | インベントリ内でAI Propの移動頻度を低減 |
| グリッチ・アダプター | グリッチ射出装置の飛距離・精度を向上 |
| 発光モジュール | 暗所照明。AI Propの擬態を無効化 |

#### 長期モチベーション
- ミッションごとに報酬（給与）が支払われ、装備強化・バッグ拡張・新エリア解放に使用。
- 「ただの掃除屋」→「装備をカスタマイズするエンジニア」へとプレイヤーの自己像が進化。
- 業務日報（リザルト画面）で成績評価。ランク・称号システムで収集欲を刺激。

### 2.7. 死亡とリスポーン

- 死は物理的かつ即物的に描く（落下、圧殺、射出による衝突死など）。
- 死亡時、インベントリ内の**未浄化AI Prop**は散逸（ロスト）。Human Propは現場に残留し、回収可能。
- リスポーン地点は最後に通過した「定常化済みチェックポイント」。
- 上司から通信で嫌味が入る（「また死んだの？ 保険料こっちなんだけど」）。

## 3. レベルデザイン

### 3.1. バイオーム（3種限定）

VRAM・アセット管理の観点から、素材セットを3種に厳格に限定する。

| バイオーム | テーマ | 代表的オブジェクト |
|-----------|--------|------------------|
| **Office** | 殺風景な事務空間 | デスク、椅子、コピー機、ホワイトボード、観葉植物 |
| **Industrial** | 剥き出しの工場 | 配管、機械、コンベア、フォークリフト、ドラム缶 |
| **Nature** | テクスチャが溶けた自然 | 樹木、岩、水面、草地、動物の残骸 |

### 3.2. モジュラー生成システム

固定マップを作らず、以下のタグの組み合わせでエリア（要塞）を構築する。

**空間タグ**: `[空中]` `[地下]` `[水没]` `[無重力]` `[回転]` `[反転]`

**AI性格タグ**: `[排他的]` `[粘着]` `[擬態]` `[集団]`

**攻略タグ**: `[要缶蹴り]` `[要パルクール]` `[要ステルス]` `[要鑑定]`

**組み合わせ例**:
- `[空中]` × `[排他的]` × `Office` = 空中に浮かぶオフィス。足場が狭く、突き落としてくるAI椅子が大量にあるアスレチック。
- `[地下]` × `[擬態]` × `[要鑑定]` × `Nature` = 地下洞窟。岩に擬態したAI Propだらけ。デバッグ視界と音響キューで見破る。
- `[無重力]` × `[集団]` × `Industrial` = 無重力工場。配管の群れが包囲してくる。

### 3.3. ステージクリア（納品ゲート）

- 各エリアの最奥に**納品ゲート（Upload Port）**が存在する。
- ゲート自体がグリッチで封鎖されているため、周辺の要塞を攻略してゲートを定常化する必要がある。
- 定常化したゲートに、規定量のHuman Propまたはデータを投入（Upload）する。
- 「Upload Complete」と共にゲートが開き、次のエリア（階層）へ移動。
- リザルトは**業務日報形式**で表示（回収率、被害額、上司コメントなど）。

> **設計意図**: 探索と収集が「ノルマ達成」という労働のメタファーに直結する。

### 3.4. 主要ギミック

すべてのギミックは「バグ」ではなく「上司が認めた仕様」として世界観内に位置づける。

| ギミック | 機能 | 用途 |
|---------|------|------|
| **グリッチ射出装置（Glitch Cannon）** | 特定オブジェクトに触れると物理演算が暴走し、超高速で射出 | ファストトラベル、ショートカット、攻撃 |
| **理想の椅子（Dev Chair）** | デザイナーの情熱がこもった椅子。乗って滑走可能。物理的に安定 | 移動手段、盾、滑走アクション |
| **偽トランポリン（Mimic Pad）** | 乗ろうとするとスッと避けるAI床 | 落下トラップ、配信映え事故 |
| **だるまさんが転んだ（Freeze Tag）** | プレイヤーが見ていない時だけ動くAI Prop群 | 緊張感の演出、背後からの追い詰め |
| **ジープ射出（Jeep Launch）** | 乗り物を物理バグで超加速 | 帰還用ファストトラベル、バイラル映像素材 |

## 4. アート＆演出

### 4.1. ビジュアル・スタイル

#### AI Prop ― 二態描画

| 状態 | 描画 | 演出意図 |
|------|------|---------|
| **休眠（Dormant）** | 不気味に完璧。テクスチャは均一すぎ、ポリゴンは機械的に整いすぎている。「作り物」の薄気味悪さ。微細なシマー（表面のちらつき）。 | 「完璧すぎて不気味」＝逆チューリングテストの前提。プレイヤーに違和感を覚えさせる。 |
| **敵対（Hostile）** | グリッチ崩壊。シェーダーでノイズ、色収差、頂点のランダム揺らぎ（Vertex Displacement）を適用。うねり・融合・分裂。 | 戦闘の視覚フィードバック。「化けの皮が剥がれた」感。 |

#### Human Prop ― 温かみのある不完全さ
- わずかに歪んだ形状、手描き風テクスチャ、ポリゴンの「味」。
- 手作り陶器のような温かみ。近づくとアナログな音響キュー（§4.3参照）。

#### 定常化（Normalization）エフェクト
- AI Propを攻撃して定常化する瞬間、「パキッ」とした切り替えエフェクト。
- パーティクル＋SE（整った音）で「掃除完了」の快感を明確に演出。

### 4.2. キャラクターデザイン

#### 主人公（Player）
- **外見**: スーツを着た猫背の二足歩行猫（ファーリー）。可愛すぎない、哀愁のあるデザイン。
- **設定**: 自分を人間（社員）だと思い込んでいる。ドライで冷笑的。
- **ボイス**: 独り言（Barks）のみ。猫の鳴き声＋電子音にテキスト字幕を乗せる形式（ボイス収録コストカット）。
  - 例: 「あー、帰りてえ」「これ経費で落ちるかな」「仕様です（棒読み）」

#### ドローン（Drone）
- **外見**: 1等身のモノアイ球体。無機質。
- **機能**: トラクタービーム（保持/投擲/攻撃）、スキャンライト、デバッグ視界の投影元。
- **性格**: 無言。上司の命令を忠実に実行する「現場監督」。

#### 上司（Boss）
- **登場形式**: UI越しの通信のみ。姿は見えない。
- **役割**: コストカットの指示、理不尽なKPI通達、死亡時の嫌味、ポップアップ通知による精神的干渉。
- **演出**: プレイヤーを「猫」としてではなく「代替可能なデバッガー」として扱う。

### 4.3. サウンドデザイン（三層レイヤー）

音響を三層に分離し、ゲームプレイ情報と世界観演出を兼ねる。

#### Layer 1: 環境音（BGM/Ambient）

| ゾーン | 音響 | 意図 |
|--------|------|------|
| **AI汚染エリア** | BGMなし。不気味に完璧だがサンプリングレートがわずかにおかしい環境音。時折デジタルノイズ。敵対時はグリッチ音が激化。 | 不安の醸成。「聴覚的アンキャニーバレー」。 |
| **定常化エリア（安全地帯）** | 安っぽいエレベーターミュージック（Bossa Nova等）や企業PRビデオ風のフリーBGM。 | 「正常＝退屈な管理社会」の皮肉。曲は著作権フリーのストック音源で賄う。 |

#### Layer 2: 近接キュー（索敵補助）

| 対象 | 音 | 機能 |
|------|---|------|
| **Human Prop 付近** | 暖かみのあるアナログノイズ（レコードのプチプチ音、暖炉のような低周波） | スキャンの代替。耳で「人間製」を探せる。 |
| **AI Prop（擬態中）付近** | 微かなデジタルハム音 | 擬態を音で見破るヒント。 |

#### Layer 3: フィードバック音

| イベント | SE |
|---------|-----|
| 定常化成功 | 「パキッ」＋整った和音。掃除完了の快感。 |
| Human Prop 発見 | 暖かいチャイム。 |
| 死亡 | 事務的な「業務終了」チャイム。皮肉。 |
| 上司通信 | 安いインターホン音。 |

## 5. UI/UXデザイン

### 5.1. UI階層

| モード | 表示内容 |
|--------|---------|
| **通常視界** | ゲーム画面＋最小限のHUD（HP、ドローンチャージ、ミニマップ） |
| **デバッグ視界** | ワイヤーフレーム＋オブジェクトID＋コンポーネントスロット表示＋AI/Human識別ハイライト |
| **インベントリ** | グリッドUI＋アイテム詳細＋コンポーネント装着パネル |
| **上司ポップアップ** | ゲーム中に強制表示される通知。KPI、嫌味、業務命令。精神的干渉の演出。 |

### 5.2. リザルト画面（業務日報）

- フォーマット: 企業の業務報告書風レイアウト。
- 表示項目: 回収Prop数/種類、定常化率、被害額（死亡・破損コスト）、上司評価コメント、給与明細。
- ランク: S/A/B/C/D。ランクに応じたボーナス。

## 6. オンボーディング（チュートリアル）

### 6.1. ファースト・コンタクトの儀式

開始5分以内に、言葉ではなく「現象」でルールを教え込む。テキスト説明は一切使わない。

#### シーケンス

1. **起動**: 猫がオフィスの椅子に座っている。上司の通信が入り、デバッグ業務を指示。
2. **最初の投擲**: 目の前に石（Human Prop）がある。ドローンで拾い、木（AI Prop・休眠状態）に投げる。
3. **世界の反撃**: 木が「殴り返してくる」。休眠状態から敵対状態へ移行する様を体験。
4. **定常化体験**: もう一度攻撃すると木が定常化。「パキッ」というSEと共に静止オブジェクトに変化。掃除の快感を初体験。
5. **逆チューリングテスト**: 複数のオブジェクトが並ぶ部屋。デバッグ視界を起動し、1つだけ混じったHuman Propを発見して回収。
6. **納品**: ゲートにHuman Propを投入。業務日報が表示される。ループの全体像を体感。

> **原則**: 最初の5分で「攻撃→反撃→定常化→鑑定→回収→納品」のフルサイクルを非言語的に体験させる。

## 7. マーケティング＆バイラル戦略

### 7.1. バイラル・フック

「リアクションを取りやすい事故」を序盤に意図的に集中させ、ストリーマーによる拡散を狙う。

| フック | 内容 | バイラル性 |
|--------|------|-----------|
| ジープ射出事故 | ファストトラベルのつもりが制御不能で壁に激突 | 物理コメディの定番。クリップ映え |
| インベントリ暴走 | AI Propが暴れて貴重品を弾き出す | パズル失敗の悲鳴 |
| 偽トランポリン | 信じて飛び乗ったら避けられて落下死 | 初見殺しの笑い |
| だるまさんが転んだ | 振り向いたら背後に木の群れ | ホラー×コメディの緊張感 |

### 7.2. メタ的文脈

「AIが作ったゴミを掃除する」という構造が、AI技術に対する現代人の不安や皮肉をエンターテイメントとして昇華する。Z世代および配信市場をメインターゲットとする。

### 7.3. バーティカル・スライス目標

15秒の「バズる映像」が撮れる状態を最初のマイルストーンとする。

## 8. 技術アーキテクチャ＆リスク軽減

### 8.1. エンジン・基盤

- **エンジン**: Unity（PhysX）
- **物理原則**: すべての挙動は物理ベース。アニメーションで誤魔化さず、ColliderとRigidbodyで制御する。
- **プロシージャルアニメーション**: AI Propの「うねり」「断末魔」は、ボーンアニメーションではなく、シェーダーやスクリプトによる頂点操作/スケール操作で実装（コスト削減）。

### 8.2. 技術リスクと対策

| # | リスク | 深刻度 | 対策 |
|---|--------|--------|------|
| 1 | **地形貫通**: AI生成メッシュのNon-manifold構造により、プレイヤーやオブジェクトが地形を突き抜ける | 極高 | 視覚メッシュと物理コリジョンを**完全分離**。不可視の単純幾何（Box/Capsule）をコリジョンとして自動生成するパイプラインを構築。「溶けた見た目」と「歩ける判定」を分離する。 |
| 2 | **物理演算CPU負荷**: 多数のAI Propが一斉に物理行動を取るとフレームレート崩壊 | 高 | **アクティブ・ポーリング**: プレイヤーの視界内/一定距離内のオブジェクトのみを物理演算対象にする（それ以外はキネマティック状態）。必要な瞬間のみ物理エンジンに渡す**ハイブリッド制御**。 |
| 3 | **インベントリ内アイテム消失**: コンテナ内物理のトンネリング | 高 | インベントリはグリッドベース疑似物理で実装（§2.5で決定済み）。 |
| 4 | **VRAM不足**: 未最適化の生成AIアセットによるメモリ圧迫 | 中 | **Decimation自動化**: AI生成直後にポリゴンを1/10に削減し、LODを3段階自動作成するツールチェーンを構築。3種バイオーム限定による素材共通化を**死守**。 |

### 8.3. 最優先技術課題

1. **AI生成アセット → ゲーム用アセット自動コンバーター**の開発（Decimation + コリジョン自動生成 + LOD作成）
2. アクティブ・ポーリングによる物理演算負荷制御の検証
3. グリッドベース・インベントリのプロトタイプ

## 付録B: 旧GDDからの削除・変更項目

| 旧仕様 | 変更内容 | 理由 |
|--------|---------|------|
| Human Prop = 「既製品・高品質」（0.1-1） | → 「手作り感・温かみ」に変更 | 逆チューリングテストとの整合性。「完璧なAI品 vs 不完全な人間品」の価値逆転を明確化。 |
| AI Prop = 単一の歪んだ外見（0.1-1） | → 二態（休眠/敵対）に拡張 | 0.1-2の「高精細だがゴミ」と0.1-1の「グリッチ」を両立。ゲームプレイ上の視覚フィードバックも確保。 |
| シールド装置を「破壊」（0.1-1） | → 「缶蹴り」で弾き飛ばす | 物理演算ゲームとしての一貫性。破壊よりも物理インタラクションの方が面白い。 |

| 音響の二値切替（0.1-1） | → 三層レイヤー設計 | 0.1-2の「索敵としての音」を統合。ゲームプレイ情報と演出の両立。 |
